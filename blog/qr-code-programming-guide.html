2. What Is a QR Code?

A QR code (Quick Response code) is a type of matrix barcode invented in 1994 by Denso Wave, a Toyota subsidiary. Unlike traditional barcodes that store data horizontally, QR codes store data both horizontally and vertically, allowing them to hold significantly more information — up to 7,000 numeric characters or about 4,300 alphanumeric characters.

QR codes support four data modes:

  • Numeric: 0–9 digits (maximum density)
  • Alphanumeric: Digits, uppercase letters, and select symbols
  • Byte/Binary: Any 8-bit byte sequence (UTF-8 text, URLs, etc.)
  • Kanji: Shift JIS encoded characters

Modern QR codes also support error correction at four levels — L (7%), M (15%), Q (25%), and H (30%) — meaning even if part of the code is damaged or obscured, it can still be scanned successfully. The GadgetHumans QR Code API lets you set the error correction level with a simple parameter.

5. Generate QR Codes with cURL

cURL is the simplest way to test the API and is available on virtually every system. Here's how to generate a QR code that links to the GadgetHumans API documentation:

curl -o qr-code.png "https://api.gadgethumans.com/qr?data=https://api.gadgethumans.com/qr&format=png&size=500"

This saves the QR code image directly to qr-code.png. Let's break down the parameters:

  • data — the URL or text we're encoding
  • format=png — outputs a PNG image
  • size=500 — 500×500 pixel square

Custom Colors with cURL

You can customize foreground and background colors using hex values (without the # prefix when URL-encoded):

curl -o branded-qr.png \
  "https://api.gadgethumans.com/qr?data=https://www.gadgethumans.com/services/&format=png&foreground=%23ff6600&background=%23f0f0f0&size=400"

SVG Output with cURL

For scalable vector graphics that look crisp at any size:

curl -o qr-code.svg \
  "https://api.gadgethumans.com/qr?data=https://api.gadgethumans.com/qr&format=svg&error_correction=H"

The SVG format is ideal for web use and print media where resolution independence matters. Setting error_correction=H ensures the code remains scannable even with significant logo overlay or damage.

6. Generate QR Codes with Python

Python is the go-to language for automation and data processing. Here's how to generate QR codes using the GadgetHumans QR Code API with the requests library.

Basic QR Code Generator

import requests

def generate_qr(data, filename="qrcode.png", size=300, format="png"):
    """
    Generate a QR code using the GadgetHumans API.

    Args:
        data (str): The content to encode (URL or text)
        filename (str): Output filename
        size (int): Image size in pixels
        format (str): Output format (png, svg, jpg)
    """
    url = "https://api.gadgethumans.com/qr"
    params = {
        "data": data,
        "format": format,
        "size": size
    }

    response = requests.get(url, params=params, stream=True)

    if response.status_code == 200:
        with open(filename, "wb") as f:
            f.write(response.content)
        print(f"✅ QR code saved as {filename}")
    else:
        print(f"❌ Error {response.status_code}: {response.text}")

# Example: Generate a QR code linking to the free API
generate_qr("https://api.gadgethumans.com/qr", "my-qr.png", 500)

Python Bulk Generator

import requests
import csv
from concurrent.futures import ThreadPoolExecutor, as_completed

def generate_single_qr(data, output_dir="qr_codes", size=300):
    """Generate one QR code and save to disk."""
    url = "https://api.gadgethumans.com/qr"
    params = {"data": data, "format": "png", "size": size}

    response = requests.get(url, params=params)
    if response.status_code == 200:
        # Use a safe filename derived from the data
        safe_name = data.replace("https://", "").replace("/", "_").replace(".", "-")
        filename = f"{output_dir}/{safe_name}.png"
        with open(filename, "wb") as f:
            f.write(response.content)
        return filename
    return None

# Read URLs from a CSV and generate QR codes in parallel
urls = []
with open("urls.csv", "r") as f:
    reader = csv.reader(f)
    for row in reader:
        urls.append(row[0])

with ThreadPoolExecutor(max_workers=5) as executor:
    futures = [executor.submit(generate_single_qr, url) for url in urls]
    for future in as_completed(futures):
        result = future.result()
        if result:
            print(f"Generated: {result}")

This parallel approach can generate hundreds of QR codes in seconds — ideal for migrating a product catalog or generating QR codes for an entire event's ticket inventory.

7. Generate QR Codes with JavaScript (Node.js)

For developers building web applications, serverless functions, or Electron apps, here's how to integrate the GadgetHumans QR Code API with Node.js.

Using Node.js Native HTTPS (No Dependencies)

const https = require("https");
const fs = require("fs");

function generateQR(data, filename = "qrcode.png", size = 300) {
    const params = new URLSearchParams({
        data: data,
        format: "png",
        size: size.toString()
    });

    const url = `https://api.gadgethumans.com/qr?${params}`;

    https.get(url, (response) => {
        if (response.statusCode !== 200) {
            console.error(`Error: ${response.statusCode}`);
            return;
        }

        const fileStream = fs.createWriteStream(filename);
        response.pipe(fileStream);

        fileStream.on("finish", () => {
            fileStream.close();
            console.log(`✅ QR code saved as ${filename}`);
        });
    }).on("error", (err) => {
        console.error(`Request failed: ${err.message}`);
    });
}

// Generate a QR code for the GadgetHumans free API
generateQR("https://api.gadgethumans.com/qr", "api-qr.png", 500);

Using Axios (Modern Approach)

const axios = require("axios");
const fs = require("fs");
const path = require("path");

async function generateQRCode(options = {}) {
    const {
        data = "https://api.gadgethumans.com/qr",
        format = "png",
        size = 300,
        errorCorrection = "M",
        foreground,
        background,
        outputDir = "."
    } = options;

    const params = { data, format, size, error_correction: errorCorrection };
    if (foreground) params.foreground = foreground;
    if (background) params.background = background;

    try {
        const response = await axios.get("https://api.gadgethumans.com/qr", {
            params,
            responseType: "stream"
        });

        const ext = format === "svg" ? "svg" : format === "jpg" ? "jpg" : "png";
        const safeName = data.replace(/[^a-zA-Z0-9]/g, "_").slice(0, 40);
        const filename = path.join(outputDir, `${safeName}.${ext}`);

        const writer = fs.createWriteStream(filename);
        response.data.pipe(writer);

        return new Promise((resolve, reject) => {
            writer.on("finish", () => resolve(filename));
            writer.on("error", reject);
        });
    } catch (error) {
        console.error("QR generation failed:", error.message);
        throw error;
    }
}

// Example usage
(async () => {
    const result = await generateQRCode({
        data: "https://www.gadgethumans.com/services/",
        size: 600,
        errorCorrection: "Q",
        foreground: "#4a90d9",
        background: "#ffffff"
    });
    console.log(`✅ Saved: ${result}`);
})();

Client-Side JavaScript (Browser)

async function generateQRCodeBrowser(data, format = "png", size = 300) {
    const params = new URLSearchParams({ data, format, size });
    const url = `https://api.gadgethumans.com/qr?${params}`;

    try {
        const response = await fetch(url);

        if (!response.ok) {
            throw new Error(`HTTP ${response.status}`);
        }

        const blob = await response.blob();
        const imgUrl = URL.createObjectURL(blob);

        // Display on page
        const img = document.createElement("img");
        img.src = imgUrl;
        img.alt = `QR code for ${data}`;
        document.body.appendChild(img);

        // Create download link
        const a = document.createElement("a");
        a.href = imgUrl;
        a.download = `qrcode.${format === "svg" ? "svg" : "png"}`;
        a.textContent = "Download QR Code";
        document.body.appendChild(a);
    } catch (error) {
        console.error("Failed to generate QR code:", error);
    }
}

// Usage in the browser

8. Customizing Your QR Codes

Plain black-and-white QR codes work, but branded QR codes attract more scans. The GadgetHumans QR Code API supports several customization options to help your codes stand out.

Color Customization

Match your brand colors using the foreground and background parameters. Always ensure sufficient contrast — a dark foreground on a light background scans best. The API accepts hex color codes like #ff6600 or #333333.

Accessibility Tip: The Web Content Accessibility Guidelines (WCAG) recommend a minimum contrast ratio of 3:1 for non-text elements. For QR codes, dark foreground on light background (e.g., #000000 on #ffffff) is the most reliable combination. Avoid light-on-dark QR codes as many scanners struggle with inverted colors.

Error Correction Level

Higher error correction allows more of the QR code to be damaged or obscured while remaining scannable. Use error_correction=H (30%) if you plan to overlay a logo in the center of the QR code. Use error_correction=L (7%) for maximum data density when no logo is needed.

Size and Margins

The size parameter controls the output image dimensions in pixels. For print materials, use at least 500px; for digital display, 200–300px is usually sufficient. The margin parameter controls the quiet zone — the blank border that helps scanners identify the code. The QR standard requires at least 4 modules of quiet zone.

9. Bulk QR Code Generation

One of the most powerful use cases for programmatic QR generation is producing codes at scale. Here are several strategies for bulk generation using the GadgetHumans QR Code API.

Shell Script with cURL

#!/bin/bash
# bulk-generate.sh — Generate QR codes from a URL list

INPUT_FILE="urls.txt"
OUTPUT_DIR="qr_codes"
mkdir -p "$OUTPUT_DIR"

while IFS= read -r url; do
    # Create a safe filename
    safe_name=$(echo "$url" | sed 's/https\?:\/\///' | tr '/.?&=' '_')
    echo "Generating QR for: $url"
    curl -s -o "${OUTPUT_DIR}/${safe_name}.png" \
        "https://api.gadgethumans.com/qr?data=${url}&format=png&size=400"
done < "$INPUT_FILE"

echo "✅ Bulk generation complete. Files saved in $OUTPUT_DIR/"

Python with ThreadPoolExecutor

Refer to the Python bulk generator example in Section 6. For maximum throughput, use ThreadPoolExecutor with 5–10 concurrent workers to parallelize API calls without overwhelming rate limits.

Node.js Promise.all

const axios = require("axios");
const fs = require("fs");
const path = require("path");

async function bulkGenerate(urls, outputDir = "./qr_codes") {
    fs.mkdirSync(outputDir, { recursive: true });

    const results = await Promise.allSettled(
        urls.map(async (url) => {
            const params = { data: url, format: "png", size: 400 };
            const response = await axios.get("https://api.gadgethumans.com/qr", {
                params,
                responseType: "arraybuffer"
            });

            const safeName = url.replace(/[^a-zA-Z0-9]/g, "_").slice(0, 40);
            const filePath = path.join(outputDir, `${safeName}.png`);
            fs.writeFileSync(filePath, response.data);
            return filePath;
        })
    );

    results.forEach((r, i) => {
        if (r.status === "fulfilled") console.log(`✅ ${r.value}`);
        else console.error(`❌ Item ${i} failed:`, r.reason.message);
    });
}

10. Best Practices & Error Handling

Follow these best practices to ensure your QR codes are reliable, scannable, and production-ready.

Always Validate Input

QR codes with excessively long data may fail to generate. The GadgetHumans QR Code API handles up to 4,296 alphanumeric characters — but for the best scanning experience, keep URLs under 1,000 characters. Use URL shorteners for long URLs.

Test on Multiple Devices

Not all QR scanners are created equal. Test your generated QR codes on iOS (Camera app), Android (Google Lens), and third-party scanner apps before deploying to production.

Provide Fallbacks

Always include human-readable text or a short URL alongside QR codes in print materials. Not all users have a QR scanner, and some environments (low light, reflective surfaces) make scanning difficult.

Error Handling in Code

import requests
from requests.exceptions import RequestException, Timeout

def safe_generate_qr(data, retries=2):
    url = "https://api.gadgethumans.com/qr"
    params = {"data": data, "format": "png", "size": 300}

    for attempt in range(retries + 1):
        try:
            response = requests.get(url, params=params, timeout=10)

            if response.status_code == 429:
                print("Rate limited. Waiting before retry...")
                time.sleep(5)
                continue

            response.raise_for_status()
            return response.content

        except Timeout:
            print(f"Timeout on attempt {attempt + 1}")
            if attempt == retries:
                raise
        except RequestException as e:
            print(f"Request failed: {e}")
            if attempt == retries:
                raise

    return None

Monitor Your API Usage

Track your request count against the daily limit. The response headers include X-RateLimit-Remaining and X-RateLimit-Reset so you can proactively manage usage.

  • Cache aggressively: QR codes for static URLs (e.g., your homepage) should be generated once and cached.
  • Use SVG for web: SVG files are smaller, resolution-independent, and can be styled with CSS.
  • Choose the right error correction: Use M (15%) for general use, H (30%) for codes with logos.
  • Keep the quiet zone: Never place text or graphics within 4 modules of the QR code edges.
