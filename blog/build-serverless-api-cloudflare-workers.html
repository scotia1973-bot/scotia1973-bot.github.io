How to Build and Deploy a Serverless API on Cloudflare Workers (2026 Guide)

Cloudflare Workers is one of the cheapest, fastest ways to deploy an API. I built two production APIs with it — a QR Code Generator and an Email Verification service — without spending a cent on infrastructure.

Here's exactly how I did it.

Why Cloudflare Workers?

  • Free tier: 100,000 requests/day for free
  • Global: Deployed to 330+ data centers
  • Fast: Cold starts in microseconds
  • Simple: One JavaScript file = one API
  • No servers: No EC2, no Kubernetes, no Docker

Step 1: Set Up Wrangler

npm install -g wrangler
wrangler login

This connects your terminal to your Cloudflare account.

Step 2: Create a Worker

npx wrangler init my-api
cd my-api

This creates a minimal worker with a single index.js file.

Step 3: Write Your API

Here's a minimal QR code API in under 50 lines:

export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    const text = url.searchParams.get('text') || 'https://example.com';
    
    // Generate QR code matrix
    const qr = generateQRCode(text);
    
    return new Response(qr.toSVG(), {
      headers: { 'Content-Type': 'image/svg+xml' }
    });
  }
};

Step 4: Deploy

wrangler deploy

Your API is live on *.workers.dev in under 30 seconds.

Step 5: Add a Custom Domain

# wrangler.toml
routes = [
  { pattern = "api.yourdomain.com", zone_id = "your_zone_id" }
]

Pro Tips

  • Use KV for storage: Great for rate limiting, API keys, and session data
  • Use Secrets: Store API keys and tokens via wrangler secret put
  • Stripe integration: Workers can handle Stripe webhooks for payment processing
  • CORS headers: Always set Access-Control-Allow-Origin: * for public APIs

Cost Breakdown

  • Cloudflare Workers: $0/mo (free tier covers 100K requests/day)
  • Custom domain: Free with Cloudflare DNS
  • Wrangler CLI: Free
  • Total: $0 to build and deploy

If you're building an API, start with Cloudflare Workers. You can't beat free, fast, and simple.
