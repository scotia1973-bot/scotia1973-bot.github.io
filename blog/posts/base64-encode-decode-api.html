June 7, 2026

Introduction

Base64 encoding is fundamental to data transport on the web — used in JSON payloads, URL parameters, email attachments, and API requests. The GadgetHumans Base64 Encode Decode API provides a simple, no-auth endpoint for both encoding text to Base64 and decoding Base64 back to plain text.

API Endpoint

GET https://api.gadgethumans.com/base64

Parameters

ParameterRequiredDefaultOptions
textYes—String to encode or decode
modeNoencodeencode, decode

Examples

Encode to Base64

curl "https://api.gadgethumans.com/base64?text=Hello+World&mode=encode"
# Returns {"original": "Hello World", "result": "SGVsbG8gV29ybGQ=", "mode": "encode"}

Decode from Base64

curl "https://api.gadgethumans.com/base64?text=SGVsbG8gV29ybGQ=&mode=decode"
# Returns {"original": "SGVsbG8gV29ybGQ=", "result": "Hello World", "mode": "decode"}

MCP Tool: encode_base64

For AI agents using the MCP server, the encode_base64 tool provides the same functionality. Call it via the Streamable HTTP endpoint at https://api.gadgethumans.com/mcp or install the PyPI package.

Use in AI Agent Workflows

  • Decoding Base64-encoded API responses
  • Encoding data for transport in JSON
  • Processing image data as Base64 strings
  • Debugging encoded payloads
