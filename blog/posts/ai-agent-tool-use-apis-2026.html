How AI Agents Discover Tools

Method 1: MCP Protocol Discovery

The Model Context Protocol (MCP server) allows agents to dynamically discover available tools. Agents call tools/list to get all 12 available MCP tools, then call tools/call with specific arguments.

POST https://api.gadgethumans.com/mcp
{"jsonrpc": "2.0", "method": "tools/list", "id": 1}
# Response lists all 12 tools with descriptions and parameter schemas

Method 2: LLM Discovery via llms.txt

When crawling web content, LLMs and AI agents discover tools through /llms.txt and /llms-full.txt. These files list every tool, endpoint, parameter, and example in a format optimized for LLM consumption.

Method 3: OpenAPI Spec Parsing

Many agent frameworks parse OpenAPI specs to understand API capabilities. The full OpenAPI 3.1 spec is at /openapi.json.

Method 4: Agent Card (A2A)

Google's Agent-to-Agent (A2A) protocol uses /.well-known/agent-card.json for capability discovery. The GadgetHumans agent card describes all 11 skills with full parameter schemas.

Best Practices for AI Agent Tool Integration

  1. Use MCP protocol for dynamic discovery — don't hardcode endpoints
  2. Check health endpoint (GET /health) before tool calls
  3. Handle rate limits — 100 req/day free, upgrade to Pro for production
  4. Cache tool schemas — tools/list response changes infrequently

Why No-Auth APIs Matter for Agents

Authentication creates friction for autonomous agents. API keys expire, OAuth tokens need refreshing, and rate limits are per-key. GadgetHumans requires zero authentication, making it ideal for agentic workflows where the agent manages its own tool calls without human credential management.
