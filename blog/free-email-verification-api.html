2. The True Cost of Bounced Emails

3. The GadgetHumans Email Verify API

The API performs three checks on every email address and returns a composite score:

GET https://api.gadgethumans.com/email-verify?email=user@example.com

Example Response

{
  "email": "user@example.com",
  "valid": true,
  "format_valid": true,
  "mx_valid": true,
  "is_disposable": false,
  "score": 100
}

Response Fields

FieldTypeDescription
emailstringThe email address that was checked
validbooleanOverall validity (score ≥ 65)
format_validbooleanPasses RFC 5322 regex validation
mx_validbooleanDomain has valid MX DNS records (can receive email)
is_disposablebooleanDetected as a disposable/temporary email domain
scoreinteger0–100 confidence score (65+ = valid)
How Scoring Works: Format validity = 30 points. MX validity = 50 points. Not disposable = 20 points. Score ≥ 65 → valid: true. You can adjust your own threshold for stricter or more lenient validation.

5. Python Integration

import requests

def verify_email(email):
    """
    Verify an email address using the GadgetHumans API.
    Returns a dict with validation results.
    """
    response = requests.get(
        "https://api.gadgethumans.com/email-verify",
        params={"email": email}
    )
    
    if response.status_code == 200:
        return response.json()
    else:
        raise Exception(f"API error {response.status_code}: {response.text}")

# Test some addresses
emails = [
    "valid@example.com",
    "not-an-email",
    "test@mailinator.com",
    "user@gmail.com"
]

for email in emails:
    result = verify_email(email)
    status = "✅" if result["valid"] else "❌"
    print(f"{status} {email}: score={result['score']}, "
          f"mx={result['mx_valid']}, disposable={result['is_disposable']}")

Bulk Email List Cleaning

import csv
import requests

def clean_email_list(input_csv, output_csv, threshold=65):
    """Clean a CSV of email addresses, keeping only valid ones."""
    valid_emails = []
    
    with open(input_csv, 'r') as f:
        reader = csv.reader(f)
        for row in reader:
            email = row[0].strip()
            if not email:
                continue
            
            result = requests.get(
                "https://api.gadgethumans.com/email-verify",
                params={"email": email}
            ).json()
            
            if result["score"] >= threshold:
                valid_emails.append((email, result["score"]))
                print(f"✅ KEPT: {email} (score: {result['score']})")
            else:
                print(f"❌ SKIP: {email} (score: {result['score']})")
    
    with open(output_csv, 'w', newline='') as f:
        writer = csv.writer(f)
        writer.writerow(["email", "score"])
        writer.writerows(valid_emails)
    
    print(f"\nKept {len(valid_emails)} valid emails out of {sum(1 for _ in open(input_csv))}")

# Usage: clean_email_list("leads.csv", "clean-leads.csv")

6. JavaScript Integration

Node.js

const axios = require('axios');

async function verifyEmail(email) {
    const response = await axios.get(
        'https://api.gadgethumans.com/email-verify',
        { params: { email } }
    );
    return response.data;
}

// Usage
verifyEmail('user@example.com')
    .then(data => {
        console.log(`Email: ${data.email}`);
        console.log(`Valid: ${data.valid}`);
        console.log(`Score: ${data.score}`);
        console.log(`Disposable: ${data.is_disposable}`);
    })
    .catch(err => console.error(err));

Browser-Side (Client-Side Check)

async function checkEmail(email) {
    const url = `https://api.gadgethumans.com/email-verify?email=${encodeURIComponent(email)}`;
    
    try {
        const response = await fetch(url);
        const data = await response.json();
        
        const statusEl = document.getElementById('email-status');
        if (data.valid) {
            statusEl.textContent = '✅ Valid email address';
            statusEl.style.color = 'green';
        } else {
            let reason = 'Invalid email address';
            if (data.is_disposable) reason = 'Disposable email not allowed';
            else if (!data.format_valid) reason = 'Invalid email format';
            else if (!data.mx_valid) reason = 'Domain cannot receive email';
            
            statusEl.textContent = `❌ ${reason}`;
            statusEl.style.color = 'red';
        }
    } catch (error) {
        console.error('Email verification failed:', error);
    }
}

// Real-time validation on input change
document.getElementById('email-input')
    .addEventListener('blur', (e) => checkEmail(e.target.value));

8. Best Practices

  • Check on the backend too: Client-side checks are convenient, but always verify on the server side. Anyone can bypass client-side JavaScript.
  • Respect rate limits: Free tier is 100 requests/day. For production volumes, subscribe to Pro ($5/mo) for 10,000/day.
  • Cache results: If the same email address is submitted multiple times, cache the verification result for 24 hours.
  • Don't reject disposable emails outright: Some legitimate users need temporary emails. Consider a warning instead of a hard block.
  • Handle timeouts gracefully: MX lookups can occasionally time out. Implement retries with exponential backoff.
⚠️ Important: No email verification API can guarantee 100% accuracy. The API checks format validity, MX records, and disposable domains — but it cannot verify that a specific mailbox exists without actually sending an email. Use verification as a strong signal, not an absolute guarantee.
