I Built 3 Revenue Streams in One Weekend — Here's Exactly How (Zero Budget)

Published May 10, 2026 · GadgetHumans

Let me paint you a picture. It's Friday night. You're sitting on your couch, scrolling through Twitter, watching some SaaS founder brag about their $50k MRR. You feel that familiar mix of inspiration and dread — "I should be doing something, but where do I even start?"

But last weekend, I decided to stop thinking and start building. No investors, no funding, no fancy tools. Just a laptop, a Google Cloud account (free tier), and a stubborn refusal to overthink things.

QR codes are everywhere. Menus, business cards, product packaging, event tickets. Every business needs them, and most are either using free generators with ugly watermarks or paying $29/month for APIs they barely use.

I spotted the gap.

I built a lightweight QR Code generation API at api.gadgethumans.com. It does one thing and does it well — generate high-quality, customizable QR codes via a simple API call. PNG, SVG, custom colors, embedded logos, the works.

The pricing: $5/month for 1,000 API calls. No contracts, no tiers with features you don't need.

Here's the kicker — I built the entire thing using free Google Cloud Run instances and an open-source QR library. The domain cost me $12/year. Everything else was $0.

Stream #2: Email Verification API — $5 for Clean Lists

Every email marketer's nightmare: bounced emails. It hurts your deliverability, your sender reputation, and ultimately your bottom line.

Services like ZeroBounce and NeverBounce charge $0.01–$0.03 per verification. For a list of 10,000 emails, that's $100–$300. Every single time.

I built a better way.

My Email Verification API does the same thing — MX record checks, SMTP verification, disposable email detection, syntax validation — for a flat $5/month. Unlimited verifications within a reasonable daily cap.

It's built on open-source verification libraries, deployed on cheap cloud infrastructure, and priced so low that anyone can afford it.

The Mindset Shift That Made This Possible

Here's what I learned this weekend that applies to anyone wanting to build side income:

  1. Start smaller than you think. I didn't build a SaaS platform. I built a single API endpoint. That's it.
  2. Price for the first buyer, not the millionth. $5/month is an impulse buy. No one has to think about it.
  3. Ship before you're ready. My APIs had basic documentation. Not perfect. Good enough.
  4. Use what you already know. I didn't learn a new framework. I used Python (which I already know) and deployed it the way I've done a hundred times.
  5. Leverage free infrastructure aggressively. Cloud Run gives you 2 million requests/month free. That's enough to validate any idea.
