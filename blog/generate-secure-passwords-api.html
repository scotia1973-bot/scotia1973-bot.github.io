  • Cryptographic Quality: The GadgetHumans API uses cryptographically secure random number generation (crypto.randomBytes on Cloudflare Workers), not the weaker Math.random() that many client-side generators use.

  • Consistent Standards: Every password follows the same generation rules — no edge cases, no bugs introduced by custom implementations.

5. Generate Passwords with Python

Here's a complete Python function that integrates the password API into any application:

import requests
import json

def generate_password(
    length=16,
    numbers=True,
    symbols=True,
    uppercase=True,
    lowercase=True
):
    """
    Generate a secure password using the GadgetHumans API.
    
    Returns a dict with 'password', 'length', and 'options' keys.
    """
    url = "https://api.gadgethumans.com/password"
    params = {
        "length": length,
        "numbers": str(numbers).lower(),
        "symbols": str(symbols).lower(),
        "uppercase": str(uppercase).lower(),
        "lowercase": str(lowercase).lower()
    }
    
    response = requests.get(url, params=params)
    
    if response.status_code == 200:
        data = response.json()
        return data
    else:
        raise Exception(f"API error {response.status_code}: {response.text}")

# Example usage
result = generate_password(length=24, symbols=True)
print(f"Generated password: {result['password']}")
print(f"Length: {result['length']}")

6. Generate Passwords with JavaScript

Here's how to integrate the password API into Node.js applications and browser-based tools:

Node.js with Axios

const axios = require('axios');

async function generatePassword(options = {}) {
    const {
        length = 16,
        numbers = true,
        symbols = true,
        uppercase = true,
        lowercase = true
    } = options;

    const params = { length, numbers, symbols, uppercase, lowercase };

    try {
        const response = await axios.get(
            'https://api.gadgethumans.com/password',
            { params }
        );
        return response.data;
    } catch (error) {
        console.error('Password generation failed:', error.message);
        throw error;
    }
}

// Example
generatePassword({ length: 32, symbols: true })
    .then(result => console.log('Password:', result.password))
    .catch(err => console.error(err));

7. Password Security Best Practices

Using a secure password generator is only half the battle. Here are essential best practices for password security:

Minimum Length Matters Most

Research from NIST (National Institute of Standards and Technology) recommends passwords of at least 12–16 characters. Each additional character exponentially increases the search space for brute-force attacks:

LengthCharacter SetPossible CombinationsTime to Crack (at 10B guesses/s)
895 (all)6.6 × 10157.6 days
1295 (all)5.4 × 10231,714 years
1695 (all)4.4 × 1031140 million years
2095 (all)3.6 × 103911.4 trillion years

Character Diversity

Using all four character types (uppercase, lowercase, numbers, symbols) maximizes the entropy per character. For a 16-character password with all types, you get 9516 possible combinations — effectively uncrackable with current technology.

Never Transmit Passwords in Plain Text

Always use HTTPS (the GadgetHumans API enforces this). Store passwords using a strong hashing algorithm like bcrypt, Argon2, or PBKDF2 — never store them in plain text.

⚠️ Critical: The GadgetHumans Password API generates passwords on the server and returns them over HTTPS. Logging or caching these passwords defeats the purpose of having a secure password. Always clear sensitive data from memory and logs immediately after use.
