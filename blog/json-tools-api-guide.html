1. Introduction

JSON (JavaScript Object Notation) is the lingua franca of web APIs. It's everywhere — configuration files, API responses, database exports, webhook payloads, and state management. But working with JSON often involves tedious manual steps: is this valid? Can I format it for readability? Can I minify it for storage?

The GadgetHumans JSON Tools API handles all three operations with a single endpoint — validate, prettify, and minify. In this guide, you'll learn how to integrate JSON processing into your applications with code examples in cURL, Python, and JavaScript.

TL;DR: GET https://api.gadgethumans.com/json?json=%7B%22key%22%3A%22value%22%7D&mode=validate returns validation results, or use mode=prettify / mode=minify. Free: 100 requests/day.

2. The JSON API Endpoint

GET https://api.gadgethumans.com/json?json={your-json-string}&mode={validate|prettify|minify}

Parameters

ParameterRequiredDescription
jsonYesThe JSON string to process (URL-encoded). Max 100,000 characters.

Response Format

Each mode returns different information. You'll see the exact responses in the sections below.

3. Validate JSON

mode=validate checks whether a string is valid JSON and returns details about its structure:

curl "https://api.gadgethumans.com/json?json=%7B%22name%22%3A%22test%22%7D&mode=validate"

# Response:
# {"valid":true,"type":"object","keys":1}

For invalid JSON:

curl "https://api.gadgethumans.com/json?json=invalid-json&mode=validate"

# Response:
# {"valid":false,"error":"Invalid JSON: Unexpected token 'i' at position 0"}

Validation is useful for:

  • CI/CD pipelines: Validate configuration files before deployment
  • Form input validation: Check user-submitted JSON before processing
  • Webhook debugging: Verify that incoming webhook payloads are well-formed

4. Prettify / Format JSON

mode=prettify reformats JSON with proper indentation and sorted keys for readability:

curl "https://api.gadgethumans.com/json?json=%7B%22z%22%3A1%2C%22a%22%3A%22hello%22%2C%22nested%22%3A%7B%22x%22%3Atrue%7D%7D&mode=prettify"

# Response:
# {
#   "a": "hello",
#   "nested": {
#     "x": true
#   },
#   "z": 1
# }

The prettify mode sorts keys alphabetically and indents with 2 spaces — making it perfect for:

  • Formatting API responses for debugging
  • Creating readable configuration files
  • Preparing JSON for documentation or code reviews

5. Minify JSON

mode=minify removes all unnecessary whitespace:

curl "https://api.gadgethumans.com/json?json=%7B%0A%20%20%22name%22%3A%20%22test%22%2C%0A%20%20%22value%22%3A%2042%0A%7D&mode=minify"

# Response: {"name":"test","value":42}

Use minification for:

  • Storage optimization: Smaller JSON = less disk space and bandwidth
  • API response optimization: Minified JSON is faster to transmit
  • Cache key generation: Use minified JSON as a cache key for deterministic hashing

6. cURL Examples

Here's a quick reference for all three modes:

# Validate
curl "https://api.gadgethumans.com/json?json=%7B%22key%22%3A%22value%22%7D&mode=validate"

# Prettify (default mode — you can omit "mode")
curl "https://api.gadgethumans.com/json?json=%7B%22b%22%3A2%2C%22a%22%3A1%7D"

# Minify
curl "https://api.gadgethumans.com/json?json=%7B%0A%20%20%22key%22%3A%20%22value%22%0A%7D&mode=minify"
URL Encoding: When passing JSON as a query parameter, you must URL-encode it. Use %7B for {, %7D for }, %22 for ", etc. Most programming languages handle this automatically when using HTTP client libraries.

7. Python Integration

import requests
import json

def validate_json(json_string):
    """Check if a string is valid JSON."""
    response = requests.get(
        "https://api.gadgethumans.com/json",
        params={"json": json_string, "mode": "validate"}
    )
    return response.json()

def prettify_json(json_string):
    """Format JSON with indentation and sorted keys."""
    response = requests.get(
        "https://api.gadgethumans.com/json",
        params={"json": json_string, "mode": "prettify"}
    )
    return response.text  # Returns formatted JSON string

def minify_json(json_string):
    """Remove whitespace from JSON."""
    response = requests.get(
        "https://api.gadgethumans.com/json",
        params={"json": json_string, "mode": "minify"}
    )
    return response.text  # Returns compact JSON string

# Example usage
data = '{"z": 100, "a": ["hello", "world"], "nested": {"flag": true}}'

# Validate
result = validate_json(data)
print(f"Valid JSON: {result['valid']}")  # True
print(f"Type: {result.get('type')}")     # object

# Prettify
formatted = prettify_json(data)
print(f"\\nFormatted:\\n{formatted}")

# Minify
compact = minify_json(data)
print(f"\\nMinified: {compact}")
print(f"Size: {len(data)} chars → {len(compact)} chars")

Batch JSON Validation

import requests

def validate_json_files(file_paths):
    """Validate multiple JSON files."""
    for filepath in file_paths:
        with open(filepath, 'r') as f:
            content = f.read()
        
        result = requests.get(
            "https://api.gadgethumans.com/json",
            params={"json": content, "mode": "validate"}
        ).json()
        
        status = "✅" if result["valid"] else "❌"
        print(f"{status} {filepath}: {result}")

# Validate your config files
validate_json_files([
    "config.json",
    "manifest.json",
    "package.json"
])

8. JavaScript Integration

Node.js

async function processJSON(jsonString, mode = 'prettify') {
    const params = new URLSearchParams({
        json: jsonString,
        mode: mode
    });
    
    const response = await fetch(
        `https://api.gadgethumans.com/json?${params}`
    );
    
    if (mode === 'validate') {
        return response.json();
    }
    return response.text();
}

// Usage
(async () => {
    const data = JSON.stringify({ z: 100, a: [1, 2, 3], nested: { x: true } });
    
    const validation = await processJSON(data, 'validate');
    console.log('Valid:', validation.valid);
    
    const pretty = await processJSON(data, 'prettify');
    console.log('Pretty:', pretty);
    
    const minified = await processJSON(data, 'minify');
    console.log('Minified:', minified);
})();

9. JSON Processing Pipeline

Let's combine multiple GadgetHumans API Hub tools into a JSON processing pipeline:

import requests
import json

def process_api_response(api_url):
    """
    Fetch an API response, validate it, format it, and hash it for caching.
    Demonstrates using JSON + Hash APIs together.
    """
    # 1. Fetch the data
    response = requests.get(api_url)
    raw_data = response.text
    
    # 2. Validate the JSON
    validation = requests.get(
        "https://api.gadgethumans.com/json",
        params={"json": raw_data, "mode": "validate"}
    ).json()
    
    if not validation["valid"]:
        return {"error": "Invalid JSON response", "details": validation}
    
    # 3. Format it
    formatted = requests.get(
        "https://api.gadgethumans.com/json",
        params={"json": raw_data, "mode": "prettify"}
    ).text
    
    # 4. Hash it for caching
    hash_result = requests.get(
        "https://api.gadgethumans.com/hash",
        params={"text": raw_data, "algorithm": "sha256"}
    ).json()
    
    return {
        "valid": True,
        "type": validation.get("type"),
        "formatted": formatted,
        "sha256_fingerprint": hash_result["sha256"],
        "char_count": len(raw_data)
    }

# Process a sample
sample_json = json.dumps({
    "users": [
        {"id": 1, "name": "Alice", "role": "admin"},
        {"id": 2, "name": "Bob", "role": "user"}
    ],
    "metadata": {"version": "1.0", "count": 2}
})

result = process_api_response(sample_json)  # Passing JSON string directly
print(f"Valid: {result['valid']}")
print(f"Type: {result['type']}")
print(f"SHA-256: {result['sha256_fingerprint'][:16]}...")
print(f"Size: {result['char_count']} chars")

This pipeline demonstrates how the JSON Tools API complements other API Hub utilities — validate structure, format for readability, and hash for caching or integrity checks.
